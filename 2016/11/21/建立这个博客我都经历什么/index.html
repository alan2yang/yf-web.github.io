部署到远端

部署之前要安装一个扩展：

npm install hexo-deployer-git --save

否则会部署时会报错；

通过Git进入到Hexo-blog文件下，输入

hexo clean

hexo g

hexo d

其中hexo clean命令不用每次部署的时候都使用它，它相当于清空generate生成的文件，不是特别大的修改可以不用clean。

饶坑：一般比较大的更新本地或远端的文件时都会使用上面三条命令，但是有时候即使采用上述命令也报错。例如我遇到的git提交失败的问题，这时可以手动删除public文件夹和.deploy_git文件夹，然后再使用hexo d -g命令（是hexo g 和hexo d 的合体）
